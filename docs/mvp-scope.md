# MVP 范围定义（第二部分骨架）

## 1. 本轮必须完成（最小骨架）

### 1.1 配置与连接
- SiliconFlow API Key / 模型 / Base URL 配置与保存
- SiliconFlow 测试连接
- 本地配置持久化（`config.json`）

### 1.2 岗位搜索参数 UI 骨架
- 关键词、城市、薪资、经验、学历、分页、排序字段
- 可保存并回显搜索参数
- 可触发一次“搜索占位接口”调用

### 1.3 任务运行配置骨架
- 每日上限、发送间隔、单轮上限、模板
- Dry Run、验证码自动暂停、搜索后自动进入发送阶段（预留开关）
- 可保存并回显运行配置

### 1.4 自动求职控制台最小业务骨架
- 开始 / 暂停 / 停止状态切换
- 执行一轮骨架任务（占位搜索 + 文案预览）
- 控制台统计与日志输出

### 1.5 `mcp-boss` 接口预留层
- 进程管理：启动 / 停止 / 状态 / 日志
- 登录预留：登录状态查询、二维码请求（占位）
- 搜索预留：岗位搜索调用（占位）
- 渲染层通过 preload/IPC 可直接调用

## 2. 本轮明确不做

- 不接入真实 BOSS 扫码登录流程
- 不接入真实岗位搜索结果抓取
- 不接入真实自动打招呼发送
- 不接入历史记录、回复跟踪、多账号
- 不接入自动更新、支付/订阅、云端账号系统

## 3. 本轮验收标准

### 功能验收
- 可在 UI 中完成搜索参数与运行配置保存/回显
- 可调用登录/搜索占位接口并拿到结构化返回
- 控制台可完成状态切换、单轮执行、统计更新与日志输出
- `npm run check` 与 `npm run check:core` 均通过

### 边界验收
- 文档明确标注“当前为骨架，不包含真实 BOSS 登录/搜索/打招呼”
- 不引入 Electron 之外的新前端框架（保持原生 HTML/CSS/JS）

## 4. 下一阶段衔接

下一阶段在不改 UI 结构的前提下，将占位接口替换为真实 mcp-boss 调用：
- `mcpBoss:requestLoginQr` 接入二维码生成与轮询
- `mcpBoss:getLoginStatus` 接入登录态同步
- `mcpBoss:searchJobs` 接入真实岗位列表
- 控制台“单轮执行”接入真实筛选与发送
